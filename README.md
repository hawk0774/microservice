# Домашнее задание "Микросервисы: принципы"
| Решение | Маршрутизация запросов | Проверка аутентификации | Терминация HTTPS | Интеграция с Kubernetes | Расширяемость | Когда подходит |
|--------|------------------------|--------------------------|------------------|--------------------------|---------------|----------------|
| **Envoy / Envoy Gateway** | Гибкая маршрутизация по host / path / headers / SNI | JWT (`jwt_authn`), внешняя авторизация (`ext_authz`), интеграция с OAuth2/OIDC через auth-сервис | Да (terminate и passthrough) | Отличная поддержка через Gateway API, готов к service mesh | Фильтры, xDS, высокая гибкость | Крупные системы, высокая нагрузка, долгосрочный рост архитектуры |
| **Kong Gateway** | Маршруты и сервисы, удобная декларативная конфигурация | JWT, OAuth2, API Keys, ACL (через плагины) | Да | Kubernetes Ingress Controller | Плагины (Lua/Go), Enterprise-функции | Когда нужен готовый API-management и минимальная кастомизация |
| **Traefik** | Простая маршрутизация, автоматическое обнаружение сервисов | Middlewares, базовый JWT/OIDC, внешние плагины | Да, встроенный Let's Encrypt | Очень хорошая, cloud-native | Middlewares, плагины | Быстрый старт, простая эксплуатация в Kubernetes |
| **NGINX / NGINX Plus** | Гибкая маршрутизация через конфигурацию | JWT, OAuth2 через модули и Lua | Да, высокая производительность | Ingress Controller | Lua, сторонние модули | Классические инфраструктуры, высокий контроль и производительность |
